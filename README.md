# Mailing Service Backend
WEB-приложение с оформлением простого сайта на Django для управления рассылками сообщений клиентам, с функционалом создания, редактирования, удаления рассылок, управления получателями, отправки сообщений по требованию и аналитики.

## Описание проекта
Проект состоит из двух частей: базовый сервис для управления рассылками и расширенный функционал с регистрацией пользователей, ролями, статистикой и кешированием.

## Основные возможности
### Разработка сервиса:

Управление клиентами (получателями):

- Модель с полями Email (уникальный), Ф.И.О., комментарий;
- CRUD операции через интерфейс и API.

Управление сообщениями:

- Модель с темой и телом письма;
- CRUD операции.

Управление рассылками:

- Модель с датой начала, датой окончания, статусом (Создана, Запущена, Завершена);
- Связь с сообщением и получателями;
- CRUD операции;
- Отправка сообщений вручную через интерфейс и консольные команды.

Логирование попыток рассылок:

- Дата и время попытки, статус (Успешно/Неуспешно), ответ почтового сервера;
- Главная страница с общей статистикой по рассылкам и клиентам.

### Расширение функционала:

Работа с пользователем: 

- Регистрация, авторизация и подтверждение email;
- Восстановление пароля.

Роли пользователей:

- Пользователь: управление своими данными и рассылками, просмотр статистики;
- Менеджер: просмотр всех клиентов и рассылок, управление пользователями (блокировка);
- Статистика отправленных сообщений и успешных/неуспешных попыток;
- Кеширование на сервере и клиенте для оптимизации.

## Технологии

- Python 3.x, Django;
- Django REST Framework;
- PostgreSQL;
- Celery + Redis (асинхронные задачи отправки рассылок);
- Django Allauth / custom for аутентификации и подтверждения email;
- Docker (опционально);
- Redis (кеширование);
- SMTP для отправки email;
- JWT для API аутентификации

## Установка и запуск
Клонировать репозиторий:

```
git clone https://github.com/yourusername/mailing-service.git
cd mailing-service
```

Создать и активировать виртуальное окружение:

```
python -m venv .venv
source .venv/bin/activate  # Linux/macOS
.venv\Scripts\activate     # Windows
```

Установить зависимости:

```
pip install -r requirements.txt
```

Настроить .env файл с переменными окружения:

```
DATABASE_URL=postgres://user:password@localhost:5432/mailingdb
REDIS_URL=redis://localhost:6379/0
SECRET_KEY=your_secret_key
EMAIL_HOST=smtp.mailserver.com
EMAIL_PORT=587
EMAIL_HOST_USER=your_email
EMAIL_HOST_PASSWORD=your_password
```

Выполнить миграции и собрать статику:

```
python manage.py migrate
python manage.py collectstatic
```

Запустить сервер разработки:

```
python manage.py runserver
```

Запустить Celery и Celery Beat для очередей рассылок:

```
celery -A project_name worker -l info
celery -A project_name beat -l info
```

## Права доступа
Пользователь: CRUD для своих клиентов и рассылок, просмотр статистики своих рассылок.

Менеджер: просмотр всех клиентов и рассылок, управление пользователями (блокировка), доступ к списку всех пользователей сервиса.

Настроены ограничения доступа на уровне API и интерфейса.

## Особенности

- Валидация моделей (уникальность email, корректность статусов рассылок, логика попыток рассылок);
- Пагинация по 10 элементов в списках;
- Кеширование запросов для повышения производительности;
- Документация API доступна по /api/docs/;
- Возможность отправки email через SMTP;
- Логирование попыток рассылки с подробностями ответа сервера.
